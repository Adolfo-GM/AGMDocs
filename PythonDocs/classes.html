Python Classes

Classes are blueprints for creating objects. They allow you to group related data and functions together, making your code more organized and reusable.

What are Classes and Objects?

Think of a class as a cookie cutter and objects as the cookies. The class defines the shape and properties, while objects are the actual instances you create. For example, "Car" is a class, while "my red Toyota" is an object (instance) of that class.

Creating Your First Class

Let's start with a simple class that represents a person. Classes are defined using the class keyword.

# Define a simple class class Person: # Class attribute (shared by all instances) species = "Homo sapiens" # Constructor method (initializer) def __init__(self, name, age): # Instance attributes (unique to each object) self.name = name self.age = age # Create objects (instances) of the class person1 = Person("Alice", 25) person2 = Person("Bob", 30) # Access attributes print(f"Person 1: {person1.name}, {person1.age} years old") print(f"Person 2: {person2.name}, {person2.age} years old") print(f"Species: {person1.species}") print(f"Same species: {person2.species}")

Adding Methods to Classes

Methods are functions that belong to a class. They can access and modify the object's attributes.

class Person: def __init__(self, name, age): self.name = name self.age = age # Method to introduce the person def introduce(self): return f"Hi, I'm {self.name} and I'm {self.age} years old." # Method to have a birthday def have_birthday(self): self.age += 1 return f"Happy birthday {self.name}! Now {self.age} years old." # Method to check if person is adult def is_adult(self): return self.age >= 18 # Create and use objects alice = Person("Alice", 17) bob = Person("Bob", 25) # Call methods print(alice.introduce()) print(bob.introduce()) print(f"Alice is adult: {alice.is_adult()}") print(f"Bob is adult: {bob.is_adult()}") # Alice has a birthday print(alice.have_birthday()) print(f"Alice is now adult: {alice.is_adult()}")

A Practical Example: Bank Account

Let's create a more practical class that represents a bank account with methods to deposit, withdraw, and check balance.

class BankAccount: def __init__(self, owner, initial_balance=0): self.owner = owner self.balance = initial_balance def deposit(self, amount): if amount > 0: self.balance += amount return f"Deposited ${amount}. New balance: ${self.balance}" else: return "Invalid deposit amount" def withdraw(self, amount): if amount > 0 and amount <= self.balance: self.balance -= amount return f"Withdrew ${amount}. New balance: ${self.balance}" elif amount > self.balance: return "Insufficient funds" else: return "Invalid withdrawal amount" def get_balance(self): return f"{self.owner}'s balance: ${self.balance}" # Create bank accounts account1 = BankAccount("Alice", 1000) account2 = BankAccount("Bob") # Starts with $0 # Test the accounts print(account1.get_balance()) print(account2.get_balance()) print(account1.withdraw(250)) print(account2.deposit(500)) print(account1.withdraw(1000)) # Should fail print(account2.withdraw(100))

String Representation: __str__ Method

The __str__ method defines how your object appears when printed. This makes your objects more user-friendly.

class Student: def __init__(self, name, grade, subjects): self.name = name self.grade = grade self.subjects = subjects # This method defines how the object appears when printed def __str__(self): return f"Student: {self.name}, Grade: {self.grade}, Subjects: {', '.join(self.subjects)}" def add_subject(self, subject): if subject not in self.subjects: self.subjects.append(subject) return f"Added {subject} to {self.name}'s subjects" else: return f"{subject} is already in {self.name}'s subjects" def get_subject_count(self): return len(self.subjects) # Create students student1 = Student("Emma", "A", ["Math", "Science"]) student2 = Student("Alex", "B+", ["English", "History", "Art"]) # Print students (uses __str__ method) print(student1) print(student2) # Add subjects and print again print(student1.add_subject("Chemistry")) print(student1.add_subject("Math")) # Already exists print(f"Emma has {student1.get_subject_count()} subjects") print(student1) # Print updated student

Class Inheritance (Simple)

Inheritance allows you to create new classes based on existing ones. The new class inherits all the properties and methods of the parent class.

# Parent class (base class) class Animal: def __init__(self, name, species): self.name = name self.species = species def introduce(self): return f"I'm {self.name}, a {self.species}" def eat(self): return f"{self.name} is eating" # Child class (inherits from Animal) class Dog(Animal): def __init__(self, name, breed): # Call parent constructor super().__init__(name, "Dog") self.breed = breed # New method specific to Dog def bark(self): return f"{self.name} says Woof!" def fetch(self): return f"{self.name} is fetching the ball" # Another child class class Cat(Animal): def __init__(self, name, color): super().__init__(name, "Cat") self.color = color def meow(self): return f"{self.name} says Meow!" def purr(self): return f"{self.name} is purring contentedly" # Create objects dog = Dog("Buddy", "Golden Retriever") cat = Cat("Whiskers", "Orange") # Use inherited methods print(dog.introduce()) print(cat.introduce()) print(dog.eat()) print(cat.eat()) # Use specific methods print(dog.bark()) print(dog.fetch()) print(cat.meow()) print(cat.purr())

Key OOP Concepts
  • Encapsulation: Grouping data and methods together in a class
  • Inheritance: Creating new classes based on existing ones
  • Objects/Instances: Actual examples created from a class blueprint
  • Methods: Functions that belong to a class and can access its data
