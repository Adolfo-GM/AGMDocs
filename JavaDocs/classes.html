Classes and Objects in Java

Object-Oriented Programming (OOP) is a fundamental concept in Java. Classes are blueprints for creating objects, and objects are instances of classes that contain data (attributes) and methods (behaviors).

1. Creating a Basic Class

A class defines the structure and behavior that objects of that type will have:

// Define a simple class class Person { // Instance variables (attributes) String name; int age; String city; // Method to display person information void displayInfo() { System.out.println("Name: " + name); System.out.println("Age: " + age); System.out.println("City: " + city); } // Method to greet void greet() { System.out.println("Hello, I'm " + name + "!"); } } public class ClassExample { public static void main(String[] args) { // Create objects (instances) of the Person class Person person1 = new Person(); Person person2 = new Person(); // Set values for person1 person1.name = "Alice"; person1.age = 28; person1.city = "New York"; // Set values for person2 person2.name = "Bob"; person2.age = 35; person2.city = "Los Angeles"; // Call methods on objects System.out.println("=== Person 1 ==="); person1.greet(); person1.displayInfo(); System.out.println("\n=== Person 2 ==="); person2.greet(); person2.displayInfo(); } }

2. Constructor Methods

Constructors are special methods that initialize objects when they are created:

class Car { // Instance variables String brand; String model; int year; String color; // Default constructor public Car() { brand = "Unknown"; model = "Unknown"; year = 2000; color = "White"; } // Parameterized constructor public Car(String carBrand, String carModel, int carYear) { brand = carBrand; model = carModel; year = carYear; color = "Black"; // Default color } // Constructor with all parameters public Car(String carBrand, String carModel, int carYear, String carColor) { brand = carBrand; model = carModel; year = carYear; color = carColor; } // Method to display car information void displayDetails() { System.out.println("Car: " + year + " " + color + " " + brand + " " + model); } // Method to start the car void start() { System.out.println("The " + brand + " " + model + " is starting..."); } } public class ConstructorExample { public static void main(String[] args) { // Using default constructor Car car1 = new Car(); System.out.println("Car 1 (Default constructor):"); car1.displayDetails(); // Using parameterized constructor Car car2 = new Car("Toyota", "Camry", 2022); System.out.println("\nCar 2 (3 parameters):"); car2.displayDetails(); car2.start(); // Using constructor with all parameters Car car3 = new Car("Honda", "Civic", 2023, "Blue"); System.out.println("\nCar 3 (All parameters):"); car3.displayDetails(); car3.start(); } }

3. Encapsulation with Getters and Setters

Encapsulation hides internal details and provides controlled access through getter and setter methods:

class BankAccount { // Private instance variables (encapsulated) private String accountNumber; private String accountHolder; private double balance; // Constructor public BankAccount(String accNumber, String holder, double initialBalance) { accountNumber = accNumber; accountHolder = holder; balance = initialBalance; } // Getter methods public String getAccountNumber() { return accountNumber; } public String getAccountHolder() { return accountHolder; } public double getBalance() { return balance; } // Setter methods with validation public void setAccountHolder(String holder) { if (holder != null && !holder.trim().isEmpty()) { accountHolder = holder; } else { System.out.println("Invalid account holder name!"); } } // Method to deposit money public void deposit(double amount) { if (amount > 0) { balance += amount; System.out.println("Deposited $" + amount + ". New balance: $" + balance); } else { System.out.println("Invalid deposit amount!"); } } // Method to withdraw money public void withdraw(double amount) { if (amount > 0 && amount <= balance) { balance -= amount; System.out.println("Withdrew $" + amount + ". New balance: $" + balance); } else { System.out.println("Invalid withdrawal amount or insufficient funds!"); } } // Method to display account info public void displayAccountInfo() { System.out.println("Account Number: " + accountNumber); System.out.println("Account Holder: " + accountHolder); System.out.println("Current Balance: $" + balance); } } public class EncapsulationExample { public static void main(String[] args) { // Create a bank account BankAccount account = new BankAccount("12345", "John Doe", 1000.0); // Display initial account info System.out.println("=== Initial Account Information ==="); account.displayAccountInfo(); // Perform some transactions System.out.println("\n=== Transactions ==="); account.deposit(500.0); account.withdraw(300.0); account.withdraw(2000.0); // This should fail // Update account holder name System.out.println("\n=== Update Account Holder ==="); account.setAccountHolder("Jane Smith"); // Display final account info System.out.println("\n=== Final Account Information ==="); account.displayAccountInfo(); } }

4. Static Members

Static variables and methods belong to the class rather than instances. They can be accessed without creating objects:

class Student { // Static variable - shared by all instances static int totalStudents = 0; static String schoolName = "Java High School"; // Instance variables private String name; private int studentId; private double gpa; // Constructor public Student(String name, int studentId, double gpa) { this.name = name; this.studentId = studentId; this.gpa = gpa; totalStudents++; // Increment static variable } // Static method to get total students public static int getTotalStudents() { return totalStudents; } // Static method to get school name public static String getSchoolName() { return schoolName; } // Static method to change school name public static void setSchoolName(String newName) { schoolName = newName; } // Instance method public void displayInfo() { System.out.println("Student: " + name + " (ID: " + studentId + ", GPA: " + gpa + ")"); System.out.println("School: " + schoolName); } // Getter methods public String getName() { return name; } public int getStudentId() { return studentId; } public double getGpa() { return gpa; } } public class StaticExample { public static void main(String[] args) { // Access static members without creating objects System.out.println("School: " + Student.getSchoolName()); System.out.println("Total students: " + Student.getTotalStudents()); // Create student objects Student student1 = new Student("Alice", 101, 3.8); Student student2 = new Student("Bob", 102, 3.5); Student student3 = new Student("Charlie", 103, 3.9); System.out.println("\n=== After creating students ==="); System.out.println("Total students: " + Student.getTotalStudents()); // Display student information System.out.println("\n=== Student Information ==="); student1.displayInfo(); System.out.println(); student2.displayInfo(); System.out.println(); student3.displayInfo(); // Change school name using static method System.out.println("\n=== Changing School Name ==="); Student.setSchoolName("Advanced Java Academy"); System.out.println("New school name: " + Student.getSchoolName()); // All students now show the new school name System.out.println("\n=== Updated Student Info ==="); student1.displayInfo(); } }

Key OOP Concepts:
  • Class: A blueprint or template for creating objects
  • Object: An instance of a class with specific values
  • Constructor: Special method that initializes objects
  • Encapsulation: Hiding internal details using private variables
  • Static Members: Variables and methods that belong to the class
  • Instance Members: Variables and methods that belong to objects

5. Best Practices

  • Use meaningful class and variable names
  • Make instance variables private and use getters/setters
  • Initialize variables in constructors
  • Use static members for class-level data and utility methods
  • Keep classes focused on a single responsibility
  • Add validation in setter methods and constructors
  • Use this keyword when parameter names match instance variables
