Get Started with Java

Welcome to the Java setup guide! Follow these step-by-step instructions to install the Java Development Kit (JDK) and an IDE on your computer.

Step 1: Install Java Development Kit (JDK)

The JDK includes everything you need to develop Java applications: the compiler, runtime environment, and development tools.

For Windows:

  1. Go to Oracle JDK Downloads or Eclipse Temurin (OpenJDK)
  2. Download the Windows x64 Installer (.msi file)
  3. Run the installer and follow the setup wizard
  4. Accept the license agreement and choose installation directory
  5. Complete the installation

For macOS:

  1. Download JDK from Oracle or Eclipse Temurin
  2. Open the downloaded .dmg file
  3. Double-click the .pkg installer and follow instructions
  4. Enter your password when prompted

Alternative using Homebrew:

brew install openjdk

Step 2: Verify Java Installation

Let's make sure Java was installed correctly:

  1. Open Terminal (macOS/Linux) or Command Prompt (Windows)
  2. Type the following command and press Enter:
java -version

Step 3: Choose and Install an IDE

While you can write Java in any text editor, an Integrated Development Environment (IDE) makes development much easier.

Recommended IDEs:

  • IntelliJ IDEA Community Edition (Free): jetbrains.com/idea
  • Eclipse IDE (Free): eclipse.org/downloads
  • Visual Studio Code (Free) with Java Extension Pack: code.visualstudio.com

For beginners, we recommend IntelliJ IDEA Community Edition as it's user-friendly and powerful.

Step 4: Test Your Setup

Let's create your first Java program to make sure everything works!

  1. Create a new file called "HelloWorld.java"
  2. Type the following code:
public class HelloWorld { public static void main(String[] args) { System.out.println("Hello, Java World!"); System.out.println("Java is working!"); } }

Troubleshooting

If you encounter any issues:

  • Make sure JAVA_HOME environment variable is set correctly
  • Verify that Java is added to your system PATH
  • Try restarting your computer after installation
  • Make sure the filename matches the class name exactly (case-sensitive)
  • Check that you're in the correct directory when compiling and running
