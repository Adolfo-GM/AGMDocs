Methods in Java

Methods are blocks of code that perform specific tasks. They help organize code, promote reusability, and make programs more modular and easier to maintain.

1. Basic Method Structure

A method in Java has the following structure:

public class MethodExample { // Method without parameters and return value public static void sayHello() { System.out.println("Hello, World!"); System.out.println("Welcome to Java methods!"); } // Main method public static void main(String[] args) { System.out.println("Calling the sayHello method:"); sayHello(); // Method call sayHello(); // Can call multiple times } }

2. Methods with Parameters

Methods can accept parameters (inputs) to work with different data:

public class MethodParameters { // Method with one parameter public static void greetPerson(String name) { System.out.println("Hello, " + name + "!"); } // Method with multiple parameters public static void introduce(String name, int age, String city) { System.out.println("My name is " + name); System.out.println("I am " + age + " years old"); System.out.println("I live in " + city); } // Method with different data types public static void displayInfo(int number, double price, boolean available) { System.out.println("Product ID: " + number); System.out.println("Price: $" + price); System.out.println("Available: " + available); } public static void main(String[] args) { greetPerson("Alice"); greetPerson("Bob"); System.out.println("\n--- Introduction ---"); introduce("Charlie", 25, "New York"); System.out.println("\n--- Product Info ---"); displayInfo(101, 29.99, true); } }

3. Methods with Return Values

Methods can return values to the calling code using the return statement:

public class ReturnMethods { // Method that returns an integer public static int add(int a, int b) { return a + b; } // Method that returns a double public static double calculateArea(double length, double width) { return length * width; } // Method that returns a String public static String getFullName(String firstName, String lastName) { return firstName + " " + lastName; } // Method that returns a boolean public static boolean isEven(int number) { return number % 2 == 0; } // Method with conditional return public static String getGrade(int score) { if (score >= 90) { return "A"; } else if (score >= 80) { return "B"; } else if (score >= 70) { return "C"; } else { return "F"; } } public static void main(String[] args) { // Using methods with return values int sum = add(15, 25); System.out.println("Sum: " + sum); double area = calculateArea(5.5, 3.2); System.out.println("Area: " + area); String fullName = getFullName("John", "Doe"); System.out.println("Full name: " + fullName); boolean evenCheck = isEven(42); System.out.println("Is 42 even? " + evenCheck); String grade = getGrade(85); System.out.println("Grade for 85: " + grade); // Using return values in expressions int result = add(10, 20) * 2; System.out.println("Result: " + result); } }

4. Method Overloading

Java allows multiple methods with the same name but different parameters (overloading):

public class MethodOverloading { // Overloaded methods with different number of parameters public static int multiply(int a, int b) { return a * b; } public static int multiply(int a, int b, int c) { return a * b * c; } // Overloaded methods with different parameter types public static double multiply(double a, double b) { return a * b; } // Overloaded display methods public static void display(int num) { System.out.println("Integer: " + num); } public static void display(String text) { System.out.println("String: " + text); } public static void display(boolean flag) { System.out.println("Boolean: " + flag); } public static void main(String[] args) { // Call different overloaded multiply methods System.out.println("2 * 3 = " + multiply(2, 3)); System.out.println("2 * 3 * 4 = " + multiply(2, 3, 4)); System.out.println("2.5 * 3.2 = " + multiply(2.5, 3.2)); // Call different overloaded display methods display(42); display("Hello Java"); display(true); } }

Method Components:
  • Access Modifier: public, private, protected, or default
  • Static Keyword: Makes method belong to class rather than instance
  • Return Type: void (no return) or specific data type
  • Method Name: Should be descriptive and follow camelCase
  • Parameters: Input values in parentheses
  • Method Body: Code block enclosed in curly braces

5. Best Practices

  • Use descriptive method names that indicate what the method does
  • Keep methods focused on a single task (single responsibility)
  • Limit the number of parameters (typically no more than 3-4)
  • Always return a consistent data type
  • Use method overloading when methods perform similar operations
  • Add comments to explain complex methods
