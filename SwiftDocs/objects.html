Classes and Structures

Classes and structures are general-purpose, flexible constructs that become the building blocks of your program’s code. You use them to define properties and methods to add functionality to your types.

1. Defining a Structure

Structures are defined with the struct keyword. They are value types, meaning they are copied when they are assigned to a new variable or constant, or when they are passed to a function.

struct Resolution { var width = 0 var height = 0 func description() -> String { return "The resolution is \(width) x \(height)." } } // Create an instance of the struct var someResolution = Resolution() someResolution.width = 1920 someResolution.height = 1080 print(someResolution.description()) // Structs are value types (copied) var otherResolution = someResolution otherResolution.width = 1280 print("Original: \(someResolution.width)") print("Copy: \(otherResolution.width)")

2. Defining a Class

Classes are defined with the class keyword. They are reference types. When you assign a class instance to a new variable or pass it to a function, a reference to the existing instance is used, not a copy.

class VideoMode { var resolution = Resolution() var frameRate = 0.0 var name: String? func description() -> String { return "Video mode at \(frameRate) fps." } } // Create an instance of the class let someVideoMode = VideoMode() someVideoMode.resolution.width = 1920 someVideoMode.frameRate = 30.0 print(someVideoMode.description()) // Classes are reference types let anotherVideoMode = someVideoMode anotherVideoMode.frameRate = 60.0 print("Original: \(someVideoMode.frameRate)") print("Reference: \(anotherVideoMode.frameRate)")

3. Initializers

Initializers are like special methods that you can call to create a new instance of a particular type. Swift provides a default initializer for any structure or class that provides default values for all of its properties.

struct Fahrenheit { var temperature: Double init() { temperature = 32.0 // Default freezing point } } var f = Fahrenheit() print("The default temperature is \(f.temperature)° Fahrenheit.") class SurveyQuestion { let text: String var response: String? init(text: String) { self.text = text } func ask() { print(text) } } let cheeseQuestion = SurveyQuestion(text: "Do you like cheese?") cheeseQuestion.ask() cheeseQuestion.response = "Yes, I do like cheese." print(cheeseQuestion.response!)

Choosing Between Classes and Structures

Use structures by default. Use classes when you need Objective-C interoperability or you need to control the identity of the data you're modeling.

  • Use a structure when: The primary purpose is to encapsulate a few simple data values, the instances will be copied rather than referenced, and any properties stored by the structure are also value types.
  • Use a class when: You need inheritance, type casting to check a type at runtime, deinitializers to free up resources, or when multiple references can refer to the same instance.
