Variables & Constants

In Swift, you use variables and constants to store and refer to values. The main difference is that a constant cannot be changed after it's set, while a variable can be.

1. Declaration: `let` and `var`

You declare constants with the let keyword and variables with the var keyword.

// A constant for the maximum number of login attempts. let maximumLoginAttempts = 3 // A variable to track the current number of attempts. var currentLoginAttempt = 0 // You can change a variable's value. currentLoginAttempt += 1 print("Attempt #\(currentLoginAttempt) of \(maximumLoginAttempts)") // This line would cause an error because 'let' is a constant: // maximumLoginAttempts = 4

2. Type Annotations and Inference

Swift is a type-safe language. It can usually infer the type of a variable or constant from the value you assign. This is called type inference. You can also be explicit by providing a type annotation.

// Type inference: Swift knows this is a String. let inferredString = "Hello, Swift!" // Type annotation: Explicitly declare the type. var explicitDouble: Double = 70.0 // It's useful when you don't initialize at declaration. let city: String city = "Cupertino" print(inferredString) print("The temperature is \(explicitDouble)°F") print("Welcome to \(city)!")

3. Common Data Types

Here are some of the most common data types you'll use in Swift.

Data Type Description Example
String A sequence of characters, like "hello". let name = "Taylor"
Int Integers (whole numbers), like 5 or -23. var age = 33
Double Floating-point numbers with high precision. let pi = 3.14159
Float Floating-point numbers with lower precision. let price: Float = 19.99
Bool A boolean value, either true or false. var isLoggedIn = true

4. String Interpolation

You can easily include the values of constants and variables inside a string using string interpolation: "\(yourVariable)".

let language = "Swift" var version = 5.9 let year = 2023 let message = "Welcome to \(language) version \(version), released in \(year)!" print(message)

5. Type Casting & Conversion

Swift doesn't allow you to mix types in operations. You must explicitly convert them.

let label = "The width is " let width = 94 let labelAndWidth = label + String(width) // Convert Int to String print(labelAndWidth) let pi = 3.14159 let integerPi = Int(pi) // Convert Double to Int (truncates) print("Pi as a Double: \(pi)") print("Pi as an Int: \(integerPi)")

Best Practices
  • Prefer let over var. Use constants by default and only change to a variable if you know the value needs to change. This makes your code safer.
  • Use descriptive names for your variables and constants (e.g., var userScore instead of var s).
  • Rely on type inference for clean code, but use type annotations when the type isn't obvious or when you need a specific type (like Float instead of Double).
